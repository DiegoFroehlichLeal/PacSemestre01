1. Lógica de Programação

Definição e Importância

Definição: A lógica de programação é a técnica de encadear pensamentos para atingir um determinado objetivo. É a organização e o planejamento de instruções, em uma ordem lógica, para que o computador possa executar uma tarefa específica.

Importância: É fundamental para qualquer programador, pois constitui a base para o desenvolvimento de algoritmos coesos e eficientes. Sem uma boa lógica, os programas podem se tornar confusos, ineficientes ou simplesmente não funcionar como esperado. Ela desenvolve o raciocínio analítico e a capacidade de resolver problemas de forma estruturada.

Como a Lógica é Usada para Resolver Problemas

A lógica é aplicada na resolução de problemas através da criação de algoritmos. Um algoritmo é uma sequência finita de passos bem definidos que descrevem a solução de um problema. Para usar a lógica efetivamente:

  1. Análise do Problema: Entender completamente o que precisa ser resolvido, quais são as entradas (dados) e quais são as saídas esperadas (resultados).
  2. Divisão do Problema: Quebrar problemas complexos em subproblemas menores e mais fáceis de gerenciar.
  3. Desenvolvimento do Algoritmo: Definir a sequência de passos (instruções) de forma clara e precisa.
  4. Teste e Refinamento: Verificar se o algoritmo produz o resultado esperado para diferentes entradas e corrigir quaisquer falhas lógicas.

Exemplos Simples (Algoritmos do Cotidiano)

A lógica está presente em muitas tarefas do nosso dia a dia. Veja alguns exemplos:

Como Fazer um Bolo (Simplificado):

  1. INÍCIO
  2. Separar ingredientes (farinha, ovos, açúcar, fermento, leite).
  3. Pré-aquecer o forno a 180°C.
  4. Em uma tigela, misturar ovos e açúcar.
  5. Adicionar farinha e fermento, misturando.
  6. Adicionar leite aos poucos, até a massa ficar homogênea.
  7. Despejar a massa em uma forma untada.
  8. Levar ao forno por 30-40 minutos.
  9. Verificar se está assado (espetar um palito; se sair limpo, está pronto).
  10. Retirar do forno e esperar esfriar.
  11. FIM

Trocar uma Lâmpada Queimada:

  1. INÍCIO
  2. Verificar se há uma lâmpada nova e compatível.
  3. Desligar o interruptor de energia do cômodo. (SEGURANÇA!)
  4. Pegar uma escada, se necessário.
  5. Posicionar a escada de forma segura sob a lâmpada.
  6. Subir na escada.
  7. Girar a lâmpada queimada no sentido anti-horário para removê-la.
  8. Rosquear a lâmpada nova no sentido horário até ficar firme.
  9. Descer da escada.
  10. Ligar o interruptor para testar a nova lâmpada.
  11. Se a lâmpada não acender, verificar se está bem rosqueada ou se o problema é outro. Caso contrário, a tarefa está concluída.
  12. Descartar a lâmpada queimada em local apropriado.
  13. FIM

Fluxogramas

Fluxogramas são diagramas que representam visualmente um algoritmo, mostrando o fluxo das operações e decisões. Ferramentas como o Flowgorithm são excelentes para criar e testar fluxogramas.

Símbolos Básicos:

  • Terminal (Início/Fim): Forma oval.
  • Processamento: Retângulo (ex: soma = a + b).
  • Entrada/Saída: Paralelogramo (ex: Ler valor, Escrever resultado).
  • Decisão: Losango (ex: idade >= 18?). Possui saídas "Sim" e "Não".
  • Conector: Círculo pequeno (para ligar partes do fluxograma).
  • Linha de Fluxo: Setas indicando a direção.

Exemplo Prático: Calcular a Média de Dois Números

Fluxograma:

Passos do Algoritmo (correspondentes ao fluxograma):

  1. INÍCIO
  2. Ler o primeiro número (n1)
  3. Ler o segundo número (n2)
  4. Calcular a média: media = (n1 + n2) / 2
  5. Escrever o valor da media
  6. FIM

Exercício 1: Verificar se um número é Par ou Ímpar

Crie um fluxograma que leia um número inteiro e informe se ele é par ou ímpar.

Solução (Fluxograma):

Lógica: Um número é par se o resto da sua divisão por 2 for igual a 0.

  1. INÍCIO
  2. Ler numero
  3. Se (numero % 2 == 0) Então
    • Escrever "O número é PAR"
  4. Senão
    • Escrever "O número é ÍMPAR"
  5. FIM

Exercício 2: Semáforo

Crie um fluxograma que simule um semáforo simples. Se a cor for "VERMELHO", a ação é "PARE". Se for "AMARELO", a ação é "ATENÇÃO". Se for "VERDE", a ação é "SIGA". Considere uma entrada para a cor.
