2. Introdução à Programação em C

História e Importância da Linguagem C

A linguagem C foi criada por Dennis Ritchie nos Bell Labs da AT&T em 1972. Seu principal objetivo era ser uma linguagem para o desenvolvimento do sistema operacional UNIX. Desde então, tornou-se uma das linguagens de programação mais influentes e amplamente utilizadas.

Importância:

  • Base para muitas linguagens: C++, Java, C#, Python, JavaScript e muitas outras foram influenciadas ou construídas sobre conceitos de C.
  • Performance: Permite um controle de baixoível sobre o hardware, resultando em programas muito eficientes.
  • Portabilidade: Código C pode ser compilado para rodar em diversas plataformas.
  • Uso em Sistemas Operacionais e Embarcados: Amplamente usada no kernel do Linux, Windows, macOS, e em microcontroladores.
  • Desenvolvimento de Jogos e Aplicações Gráficas: Muitas engines de jogos e softwares gráficos utilizam C/C++ pela performance.

Estrutura Básica de um Programa em C

Um programa em C típico possui a seguinte estrutura:


// Comentário de linha única: Inclusão de Bibliotecas
#include <stdio.h>  // Biblioteca padrão para entrada e saída de dados

/*
 Comentário de múltiplas linhas:
 A função main é o ponto de entrada do programa.
 Todo programa C deve ter uma função main.
*/
int main() {
    // Corpo da função main: aqui vão as instruções do programa

    // printf é uma função da biblioteca stdio.h para imprimir dados na tela
    printf("Ola, Mundo!\n"); // "\n" é um caractere especial para nova linha

    // return 0 indica que o programa terminou com sucesso
    return 0;
}
  • #include <biblioteca.h>: Diretiva de pré-processamento que inclui o conteúdo de um arquivo de cabeçalho (biblioteca). stdio.h (Standard Input/Output) contém funções como printf (para saída) e scanf (para entrada).
  • int main() { ... }: A função principal onde a execução do programa começa.
    • int: Indica que a função main retornará um valor inteiro ao sistema operacional.
    • (): Podem conter argumentos de linha de comando (avançado).
    • { ... }: Delimitam o bloco de código da função.
  • printf("texto");: Função para exibir texto ou valores na tela (console).
  • return 0;: Finaliza a função main e retorna o valor 0 (indicando sucesso) para o sistema operacional. Um valor diferente de 0 geralmente indica um erro.
  • Comentários: // para comentários de uma linha e /* ... */ para múltiplas linhas. São ignorados pelo compilador e servem para explicar o código.

Como Compilar e Executar um Programa em C

Para que um programa C seja executado, ele precisa ser:

  1. Escrito: O código fonte é escrito em um editor de texto e salvo com a extensão .c (ex: meu_programa.c).
  2. Compilado: Um compilador C (como GCC, Clang, ou os embutidos em IDEs) traduz o código fonte .c para código de máquina, gerando um arquivo executável (ex: meu_programa.exe no Windows, ou apenas meu_programa no Linux/macOS).
  3. Executado: O arquivo executável é então rodado pelo sistema operacional.

Compiladores Populares:

  • GCC (GNU Compiler Collection): Padrão em sistemas Linux, disponível para Windows (via MinGW ou Cygwin) e macOS.
    • Comando para compilar: gcc meu_programa.c -o nome_do_executavel
    • Comando para executar (Linux/macOS): ./nome_do_executavel
    • Comando para executar (Windows): nome_do_executavel.exe
  • Dev-C++: IDE gratuita para Windows que utiliza o compilador MinGW (GCC). Possui botões para "Compilar" e "Executar".
  • Code::Blocks: IDE gratuita e multiplataforma que pode usar diversos compiladores, incluindo o GCC. Também com interface gráfica para compilação e execução.
  • Outros: Visual Studio (Windows, com compilador MSVC), Clang, compiladores online (Replit, OnlineGDB).

Atividade: Instale um compilador C (como o GCC com MinGW para Windows, ou use o que já vem no seu Linux/macOS) ou um IDE (Code::Blocks ou Dev-C++). Copie o código "Olá, Mundo!" acima, salve como ola.c, compile-o e execute-o para ver a mensagem no terminal/console.

Sintaxe Básica (Comentários, Identação, Boas Práticas)

Comentários:

Já mencionados, são essenciais para a clareza.


// Este é um comentário de linha única

/*
  Este é um
  comentário de
  múltiplas linhas.
*/

Identação:

Refere-se ao uso de espaços ou tabulações no início das linhas para mostrar a estrutura hierárquica do código (dentro de funções, ifs, loops, etc.). Embora o C não exija identação para funcionar, ela é crucial para a legibilidade.

Exemplo (boa identação):


#include <stdio.h>

int main() {
    int numero = 10;
    if (numero > 5) {
        printf("Maior que 5.\n"); // Bloco do if identado
    } else {
        printf("Menor ou igual a 5.\n"); // Bloco do else identado
    }
    return 0;
}

Boas Práticas:

  • Nomes Significativos: Use nomes claros para variáveis e funções (ex: `idadeUsuario` em vez de `i`, `calcularSoma` em vez de `cs`).
  • Consistência: Mantenha um estilo de código consistente (identação, chaves, espaçamento).
  • Simplicidade: Escreva código claro e direto.
  • Modularidade: Divida programas grandes em funções menores e reutilizáveis.
  • Comente o Necessário: Explique partes complexas ou decisões de design, não o óbvio.
  • Ponto e Vírgula: Quase toda instrução em C termina com ;. É um erro comum esquecê-lo.
